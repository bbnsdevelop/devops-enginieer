#cluster high availability on-premise
High-availability (HA) on-premise servers ensure continuous service, often targeting 99.99% or higher uptime, 
by eliminating single points of failure through redundancy. Key strategies include clustering, active-passive failover, 
load balancing, and data replication across multiple hosts to handle hardware or software failures seamlessly. 

Key Components and Strategies
Clustering & Failover: A primary server handles operations, with a standby server automatically taking over during downtime.
Redundancy: Duplicate hardware, networking, and power sources are critical to prevent localized failures from causing outages.
Data Replication: Periodic or real-time replication ensures data consistency between primary and standby servers.
Load Balancing: Distributes traffic across multiple servers, common in Remote Desktop Services (RDS).
Virtualization: Using virtual machines (VMs) facilitates faster recovery and resource reallocation. 

Implementation Best Practices
Planning: Define required uptime (SLAs), Recovery Time Objectives (RTO), and Recovery Point Objectives (RPO).
Architecture: Implement multiple nodes/servers to ensure that if one fails, others in the cluster continue to function.
Monitoring: Use tools to detect failures immediately and trigger automated failover processes.
Scope: Ensure redundancy covers not just servers, but networking, storage, and application layers

#Terraform create cluster

Step 1 — Update the system

First update Ubuntu packages.

sudo apt update
sudo apt upgrade -y
Step 2 — Install required dependencies

Install tools required to add external repositories.

sudo apt install -y gnupg software-properties-common curl
Step 3 — Add the HashiCorp GPG key

This key verifies Terraform packages.

wget -O- https://apt.releases.hashicorp.com/gpg \
| sudo gpg --dearmor \
-o /usr/share/keyrings/hashicorp-archive-keyring.gpg

This downloads the signing key used by HashiCorp packages.

Step 4 — Add the HashiCorp repository
echo "deb [arch=$(dpkg --print-architecture) \
signed-by=/usr/share/keyrings/hashicorp-archive-keyring.gpg] \
https://apt.releases.hashicorp.com \
$(lsb_release -cs) main" \
| sudo tee /etc/apt/sources.list.d/hashicorp.list

This tells Ubuntu where to download Terraform packages.

Step 5 — Update apt again
sudo apt update

This loads the new repository.

Step 6 — Install Terraform
sudo apt install terraform -y

Ubuntu will download and install Terraform.

Step 7 — Verify the installation

Check the installed version:

terraform -version

Example output:

Terraform v1.14.5
on linux_amd64

This confirms Terraform is installed successfully.

Step 8 — (Optional) Enable autocomplete

Enable CLI autocomplete:

terraform -install-autocomplete

Restart the shell afterward.
